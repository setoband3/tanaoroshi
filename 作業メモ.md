## 2. Windows（Cursor）で初日にやること

### 2.1 前提ソフト

- **Node.js LTS**（例: 22.x または 20.x）— [https://nodejs.org/](https://nodejs.org/) からインストール  
- **Git**（フォルダを clone する場合）— [https://git-scm.com/](https://git-scm.com/)  
- **Cursor** — 既に利用中であればそのまま

ターミナル（PowerShell または Cursor 統合ターミナル）で確認:

```powershell
node -v
npm -v
```

### 2.2 依存関係のインストール

プロジェクトのルート（`package.json` があるフォルダ）で:

```powershell
cd C:\path\to\tanaoroshi
npm install
```

### 2.3 環境変数

ルートに **`.env`** が必要です（リポジトリには含めない想定）。

- 雛形: **`.env.example`** をコピーして `.env` にリネーム
- 開発（SQLite）の例:

```env
DATABASE_URL="file:./prisma/dev.db"
```

### 2.4 DB スキーマの反映（初回 or スキーマ変更後）

```powershell
npx prisma db push
```

（マイグレーション履歴を残したい本番寄りの運用では `npm run db:migrate` を検討。現状は `db:push` で十分なことが多いです。）

### 2.5 開発サーバー（プレビュー）

```powershell
npm run dev
```

ブラウザで **http://localhost:3000** を開く。

## 4. Ubuntu サーバーへ移行するとき（メモ）

### 4.1 大方針

1. サーバーに Node.js LTS を入れる。  
2. リポジトリ（またはフォルダ）を配置し、`npm install` → `npm run build`。  
3. **本番は PostgreSQL 推奨**。`DATABASE_URL` を PostgreSQL 用に差し替え。  
4. `prisma/schema.prisma` の `datasource db` を `provider = "postgresql"` に変更し、マイグレーションを適用（`prisma migrate deploy` など）。  
5. 常駐: **systemd** で `npm run start` を実行するか、**Docker** / **reverse proxy（nginx）** で HTTPS とポートをまとめる。

### 4.2 PostgreSQL 化のときの作業イメージ

- `.env` の例（サーバー上）:

```env
DATABASE_URL="postgresql://ユーザー:パスワード@localhost:5432/tanaoroshi?schema=public"
```

- `schema.prisma` の `provider` を `postgresql` に変更。  
- 金額フィールドは PostgreSQL 利用時に `@db.Decimal(14, 2)` などを付けられる（SQLite との両立は運用で調整）。  
- 初回: `npx prisma migrate dev`（開発）または `npx prisma migrate deploy`（本番）。

### 4.3 認証

現状は **認証なし**（社内・本人のみアクセス前提）。外向きに公開する前に、VPN・Basic 認証・またはアプリ側ログインのいずれかを検討。
